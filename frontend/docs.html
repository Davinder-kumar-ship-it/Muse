Introduction

MUSE is a token-authenticated REST API that returns mood-matched verses from Hindi, Urdu, and Punjabi poetry — along with YouTube search links for related songs.

Built entirely with Node.js core modules — no Express, no frameworks. Tokens are session-based (in-memory). Users and their hashed passwords are persisted in a local JSON file.

// Typical flow 1. POST /signup → create account 2. POST /login → get token 3. POST /muse/alldata { token, mood } → get quote + YouTube link 4. PATCH /logout → invalidate token

Base URL

http://localhost:8000

All endpoints are relative to this base URL. CORS is enabled for all origins.

Authentication

MUSE uses simple token-based auth. Log in to receive a random hex token. Pass this token in the body of protected requests. Tokens live in memory — they are cleared when the server restarts.

// Protected request body example { "token": "a1b2c3d4e5f6...", "mood": "happy" }

POST /signup

Create a new user account. Passwords are hashed with crypto.scrypt + a random 16-byte salt before storage.

POST /signup No auth required

POST /login

Authenticate and receive a session token. The token must be included in all subsequent protected requests.

POST /login No auth required
// Request { "username": "priya", "password": "mypassword" } // Response 200 { "status": 200, "token": "a3f8c2d1e9b0...", "username": "priya" }
200 OK 401 Invalid credentials

POST /muse/alldata

Fetch a random mood-matched verse. Returns the quote text, author, language, theme, vibe colors, and a YouTube search URL.

POST /muse/alldata 🔒 Token required
FieldTypeRequiredDescription
tokenstringRequiredSession token from /login
moodstringRequiredOne of: happy, sad, angry, peaceful
// Request { "token": "a3f8c2d1e9b0...", "mood": "sad" } // Response 200 { "status": 200, "emotion": "sad", "quote": { "id": 1, "text": "Hazaron khwahishen aisi...", "author": "Mirza Ghalib", "language": "Urdu", "theme": "longing", "youtube": "https://www.youtube.com/results?search_query=...", "vibe_colors": { "background": "#0f1624", "accent": "#4a90d9" } } }
200 OK 400 Invalid mood 401 Invalid token

PATCH /logout

Invalidate the current session token. The token is removed from the in-memory session store.

PATCH /logout 🔒 Token required
// Request { "token": "a3f8c2d1e9b0..." } // Response 200 { "status": 200, "message": "Logged out successfully" }
200 OK 401 Invalid token

GET /api/moods

Public endpoint. Returns the list of available mood values.

GET /api/moods Public
// Response 200 { "status": 200, "moods": ["happy", "sad", "angry", "peaceful"] }

Available Moods

Each mood has 7 verse entries drawn from Hindi, Urdu, and Punjabi poetry. A random entry is returned on each request.

😢 sad 😊 happy 😤 angry 😌 peaceful

angryHindirage, defiance, fire, storm, justice, resilience, strength
peacefulUrdu, Hindiserenity, clarity, letting go, contentment, calm, mindfulness, stillness

Response Schema

All responses include a status field matching the HTTP status code. Error responses use an error field; success responses use message or data fields.

FieldTypeDescription

emotionstringThe mood that was queried

quote.textstringVerse text
quote.authorstringPoet/author name
quote.languagestringLanguage of the verse
quote.themestringEmotional theme tag
quote.youtubestringYouTube search URL for related songs
quote.vibe_colorsobjectSuggested background & accent hex colors
